=========================

一次性：在服务器上配置 Git 和 SSH（只做一次）

=========================

查看服务器上是否安装了 git

设置 git 提交显示的名字

设置 git 提交显示的邮箱（建议用 GitHub 绑定的邮箱）

生成 SSH 密钥对，一路回车即可（默认保存在 ~/.ssh/id_ed25519）

显示公钥，复制这一整行到剪贴板

【手动操作】打开 GitHub -> Settings -> SSH and GPG keys -> New SSH key -> 粘贴公钥并保存。在 GitHub 上添加这台服务器的 SSH 公钥

测试 SSH 连接，看到 “Hi xxx! You’ve successfully authenticated” 就说明连上了

=========================

每个新项目：推到 GitHub

=========================

进入你在服务器上的项目根目录（能看到 main.py 等文件）

初始化 git 仓库（目录里会多出一个 .git 文件夹）

添加远程仓库（用 SSH 地址，比如 git@github.com:xxx/xxx.git）

可选：创建一个简单的 .gitignore，避免把数据和模型权重推上去

上面这段结束后按回车就会写入 .gitignore 文件

把当前目录下所有文件加入暂存区

提交一次（写上本次提交的说明）

把当前分支命名为 main（统一成 GitHub 默认分支名）

推送到 GitHub 的 main 分支，第一次会建立关联

=========================

日常使用：修改代码后的同步

=========================

查看当前修改了哪些文件

把所有修改加入暂存区

提交一次（写上这次改动的说明）

把本次提交推送到 GitHub 远程仓库

=========================

用完服务器准备归还时：安全收尾（建议做）

=========================

【手动操作】在 GitHub -> Settings -> SSH and GPG keys 中删除这台服务器对应的 SSH key

删除后，即使别人拿到这台机器上的私钥，也无法再访问你的 GitHub

删除服务器上的 SSH 私钥和公钥文件（保险起见）

删除项目代码（不想留在服务器上就删掉，注意路径别写错）
